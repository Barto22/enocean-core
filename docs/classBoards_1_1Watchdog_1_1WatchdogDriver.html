Detailed Description

Zephyr RTOS watchdog timer driver implementation.

Concrete implementation of the IWatchdog interface for Zephyr RTOS. This driver provides hardware watchdog functionality using Zephyr's native watchdog device API. The watchdog must be periodically fed to prevent system resets.

Note
This class is final and cannot be further derived.

Feed the watchdog timer to prevent system reset.

Resets the watchdog counter using Zephyr's watchdog feed API. This must be called periodically before the configured timeout period expires to keep the system running.

Initialize the Zephyr watchdog timer with specified timeout.

Configures and starts the hardware watchdog timer using Zephyr's device API. Once initialized, the watchdog must be fed periodically via feed_impl() before the timeout expires.
