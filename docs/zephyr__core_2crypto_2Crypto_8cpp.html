Detailed Description

Zephyr AES-128-CCM-4 implementation using bt_ccm_decrypt.

AAD construction matches the NCS sdk-nrf enocean.c implementation:

bt_ccm_decrypt(key, nonce, tag, 0, payload, tot_len, NULL, 4)

where payload is the raw BLE AD record from the length byte onward: [LEN][0xFF][0xDA][0x03][seq(4)][sensor_data / switch_state]

The 'aad' parameter from EnoceanDriver contains seq+data (the bytes after the 2-byte Company ID, up to but not including the 4-byte tag). The LEN byte is reconstructed as: LEN = 1(TYPE=0xFF) + 2(CID) + aad.size() + ct_len + k_tag_len
